# 代码质量门禁（V1）

本门禁用于降低单人审计负担，保证每次变更可解释、可验证、可回滚。

## 1. 提交流程

1. 更新模块文档（职责/边界/接口/测试影响）。
2. 完成代码实现。
3. 本地执行门禁命令。
4. 提交 PR（或等价代码评审流程）并附测试结果。

## 2. 静态门禁

必须通过：

1. `go fmt ./...`
2. `go vet ./...`
3. `golangci-lint run`
4. 依赖安全扫描（可接入 `govulncheck`）

## 3. 动态门禁

必须通过：

1. `go test ./... -race`
2. 集成测试（仿真交易所闭环）
3. 回放一致性测试（同输入同输出）
4. 延迟基准测试（P99 不回退）

可选（phase-13 起）：

5. sandbox 联调测试（`RUN_SANDBOX_TESTS=1 go test ./test/sandbox/... -v`）
6. sandbox 交易烟测（`RUN_SANDBOX_TESTS=1 RUN_SANDBOX_TRADE_TESTS=1 ... go test ./test/sandbox/... -v`）

## 4. 文档门禁

1. 新增模块必须新增模块文档。
2. 接口或事件字段变更必须更新契约文档。
3. 交易逻辑变更必须更新测试矩阵与基线结果。

## 5. 注释规范

1. 必须注释：边界约束、不变量、失败语义、性能假设。
2. 禁止注释：描述代码字面行为的无效注释。
3. 导出类型/函数需要简洁注释，说明输入输出和错误条件。

## 6. 上线门禁

1. P99 超过阈值禁止发布。
2. 订单状态机测试失败禁止发布。
3. 回放一致性失败禁止发布。
4. 未通过回滚演练的版本禁止发布。
