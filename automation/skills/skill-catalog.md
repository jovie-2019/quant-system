# Skill Catalog (V1)

## 1. `skill-spec-guard`

- 用途：校验文档中的职责边界和接口是否一致。
- 输入：需求描述、现有 `docs/services/*`。
- 输出：文档 diff、边界冲突报告。
- 通过条件：无边界冲突，无未解释的破坏性变更。

## 2. `skill-contract-guard`

- 用途：校验事件契约兼容性。
- 输入：`docs/services/contracts/*`。
- 输出：契约变更报告、兼容性结论。
- 通过条件：仅追加字段或明确标注破坏性变更并触发 HITL。

## 3. `skill-go-impl`

- 用途：在指定模块内实现代码。
- 输入：模块文档、工单边界、接口契约。
- 输出：代码 diff、实现说明、风险清单。
- 通过条件：无越权改动，编译通过。

## 4. `skill-test-matrix`

- 用途：按测试矩阵补齐与执行测试。
- 输入：`docs/services/quality/testing-matrix.md`。
- 输出：测试报告（单测/集成/回放/性能）。
- 通过条件：门禁全绿，失败项有定位与复现步骤。

## 5. `skill-latency-gate`

- 用途：执行性能基线并检测回归。
- 输入：性能基线、当前构建。
- 输出：P50/P95/P99 对比报告。
- 通过条件：不超过阈值，或有批准的豁免单。

## 6. `skill-k8s-delivery`

- 用途：生成或校验 k8s 发布配置与回滚流程。
- 输入：`deploy/k8s/*`、服务变更说明。
- 输出：部署计划、回滚计划、变更影响分析。
- 通过条件：探针、资源限制、PDB、滚动策略完整。

## 7. `skill-human-review-pack`

- 用途：把复杂执行结果压缩为可审计摘要。
- 输入：测试结果、性能结果、变更清单。
- 输出：审计包（结论、证据、待确认事项）。
- 通过条件：一页内可做 go/no-go 判断。

## 8. `skill-supervisor-report`

- 用途：将多 agent 结果压缩为“监工可读”的进展报告与拍板请求。
- 输入：工单状态、测试报告、性能报告、风险清单。
- 输出：阶段进展报告、阻塞项、需要人类决策的问题单。
- 通过条件：报告可在 2 分钟内完成 go/no-go 或补充信息决策。
