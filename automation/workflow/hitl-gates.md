# HITL Gates (Human-in-the-loop)

以下节点必须由你（唯一人类审计者）确认，agent 无权自动放行。
`supervisor-report-agent` 负责在触发时主动向你发起拍板请求。
若拍板请求超时未回复，默认 `NO-GO` 并暂停流水线。

新增前置约束：

1. 任何发布相关拍板前，必须先有一份 `MCP pass` 证据。
2. MCP 失败或证据缺失时，不允许进入 GO 讨论。

## Gate 1: 架构与边界变更

触发条件：

1. 模块职责变化
2. 跨边界调用新增
3. 事件契约破坏性变更

人类确认项：

1. 边界是否仍可审计
2. 是否引入额外系统复杂度

## Gate 2: 交易关键路径变更

触发条件：

1. `risk`、`execution`、`orderfsm`、`position` 模块改动

人类确认项：

1. 是否存在绕过风控可能
2. 状态机是否仍单一真相源

## Gate 3: 性能回归异常

触发条件：

1. `market_pipeline_latency_ms P99` 回退 > 10%
2. `strategy_to_execution_latency P99` 回退 > 10%

人类确认项：

1. 是否接受豁免
2. 是否延期发布

## Gate 4: 发布与回滚

触发条件：

1. 任何生产发布
2. 任何回滚执行
3. MCP 门禁结果为 `fail` 或证据不完整

人类确认项：

1. 发布窗口与风险可接受
2. 回滚脚本已验证可执行
3. MCP 失败项是否可接受临时豁免

## Gate 5: Agent 低置信度

触发条件：

1. 任一 agent 标记 `NEED_HUMAN_CONFIRMATION`

人类确认项：

1. 给出明确决定：批准、驳回、补充信息后重跑

## Gate 6: 计划外复杂度增长

触发条件：

1. 新增中间件、外部依赖或部署组件
2. 工单范围较初始计划扩大超过 30%

人类确认项：

1. 是否接受复杂度上升
2. 是否拆分为后续里程碑

## Gate 7: MCP 策略豁免

触发条件：

1. 临时下调 MCP 阈值
2. 以手工检查替代自动检查

人类确认项：

1. 豁免有效期
2. 风险承担人与补偿工单
3. 过期后的自动恢复策略
